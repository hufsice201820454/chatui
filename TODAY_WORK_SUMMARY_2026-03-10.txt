[Backend - MCP 구조]

  - backend/app/mcp/tools/oracle.py
  - backend/app/mcp/tools/image_parser.py
  - backend/app/mcp/tools/plus.py
- 파일: backend/app/mcp/server.py

  - oracle.register(mcp), image_parser.register(mcp), plus.register(mcp)
  - SSE 서버 실행: mcp.run(transport=\"sse\")

[Backend - Config]

3) 오늘 발생한 주요 이슈/원인/해결

이슈 A) 이미지 업로드 시 \"이미지를 분석할 수 없다\" 응답
- 원인:
  - 프론트 이미지 파트 전달 형식(blob/dataURL) 불일치 + 백엔드 수신 누락 케이스.
- 조치:
  - 프론트에서 이미지 변환/추출 로직 보강.
  - body.images 우선 수신 처리 추가.
  - 백엔드 이미지 파이프라인을 MCP 툴 경로로 통일.

이슈 B) MCP 실행 오류들
- ModuleNotFoundError / import 오류:
  - mcp 패키지와 app.mcp 경로 혼용, 심볼 import 불일치.
  - register 함수 패턴으로 정리.
- run_stdio 없음:
  - 현재 SDK 인터페이스와 예제 코드 불일치.
- Already running asyncio:
  - event loop 중첩 호출.
- FastMCP.run(host=...) TypeError:
  - 현 버전 run()은 host/port 인자를 받지 않음.
  - host/port를 FastMCP 생성자에 설정하도록 수정.

이슈 C) websockets-sansio / uvicorn 충돌
- 원인:
  - fastmcp/mcp가 요구하는 uvicorn/httpx/starlette 버전과
    FastAPI requirements 충돌.
- 현재 방향:
  - 코드 레벨은 SSE 기준으로 맞춰둠.
  - 실행 환경은 버전 충돌 주의 필요(venv 분리 권장).

4) 현재 동작 상태
- 최신 확인 기준:
  - MCP 서버는 localhost:8001에서 SSE로 기동 확인.
  - 백엔드 채팅 요청에서 has_images=1 확인.
  - 툴 호출 로그(CallToolRequest) 확인.
  - 이후 MCP 이미지 툴 실패 원인은 MCP_SSE_URL 접근 버그였고 수정 완료.
  - 최종적으로 이미지 분석 응답이 정상적으로 출력되는 화면 확인됨.
